
## 👥 חברי הצוות וחלוקת עבודה

### רז עייני
אחראי על כתיבת וניהול 5 הסקריפטים הבאים:

1. **יצירת סיסמה רנדומלית מאובטחת** (`password_generator.sh`)
2. **סריקת פורטים פתוחים (Port Scanner)** (`port_scanner.sh`)
3. **הוספת תחילית לקבצי TXT בתיקייה** (`bulk_rename.sh`)
4. **הצגת סטטוס Git לכל תת-תיקייה** (`git_status_checker.sh`)
5. **בדיקת תקינות גישה (HTTP Status) לרשימת אתרים** (`http_checker.sh`)

---

## 🚀 הוראות שכפול והרצה (Getting Started)

כדי להפעיל את הסקריפטים על המחשב המקומי שלך, יש לעקוב אחר השלבים הבאים:

### 1. שכפול הפרויקט (Clone)

פתח את הטרמינל והרץ את הפקודות הבאות:

```bash
git clone <URL_OF_YOUR_GITHUB_REPOSITORY>
cd <REPOSITORY_NAME>
```

### 2. מתן הרשאות הרצה (Execution Permissions)

כל הסקריפטים נמצאים תחת תיקיית `scripts/`.

לפני ההרצה הראשונית, יש לתת להם הרשאות הרצה באמצעות:

```bash
chmod +x scripts/*.sh
```

---

## 📄 פירוט הסקריפטים והוראות הפעלה

## 1. הצגת סטטוס Git לכל תת-תיקייה (`git_status_checker.sh`)

### מה זה עושה?

סורק את תיקיית האב, מזהה אוטומטית אילו תת-תיקיות הן מאגרי Git (מכילות תיקיית `.git`), ומציג פלט קומפקטי ומרוכז של הסטטוס שלהן (`git status -s`) מבלי לשנות את תיקיית העבודה הנוכחית.

### איך מריצים?

```bash
./scripts/git_status_checker.sh
```

### קלט נדרש בזמן הריצה

הסקריפט יבקש להזין את נתיב תיקיית האב.

לחיצה על `Enter` תבחר בתיקייה הנוכחית (`.`) כברירת מחדל.

---

## 2. יצירת סיסמה רנדומלית מאובטחת (`password_generator.sh`)

### מה זה עושה?

מייצר סיסמה חזקה ומאובטחת באורך 10 תווים הכוללת לפחות:

- אות גדולה (`A-Z`)
- אות קטנה (`a-z`)
- מספר (`0-9`)
- סימן מיוחד

הסיסמה נוצרת באמצעות `/dev/urandom`.

### איך מריצים?

```bash
./scripts/password_generator.sh
```

### קלט נדרש בזמן הריצה

אין צורך בקלט.

הסיסמה תודפס ישירות למסך.

---

## 3. סריקת פורטים פתוחים (`port_scanner.sh`)

### מה זה עושה?

בודק אילו פורטים פתוחים להאזנה בכתובת IP מוגדרת, תוך שימוש במנגנון ה-TCP המובנה של Bash (`/dev/tcp`) ללא תלות בכלים חיצוניים.

### איך מריצים?

```bash
./scripts/port_scanner.sh
```

### קלט נדרש בזמן הריצה

1. כתובת IP או Hostname לסריקה.
2. פורט התחלה.
3. פורט סיום.

---

## 4. הוספת תחילית לקבצי TXT (`bulk_rename.sh`)

### מה זה עושה?

משנה שם המוני (Bulk Rename) לכל קבצי ה-`.txt` בתוך תיקייה נתונה על ידי הוספת קידומת (Prefix) קבועה לבחירת המשתמש.

### איך מריצים?

```bash
./scripts/bulk_rename.sh
```

### קלט נדרש בזמן הריצה

- הנתיב המלא לתיקייה שבה נמצאים הקבצים.
- התחילית הרצויה (לדוגמה: `backup_`).

---

## 5. בדיקת תקינות גישה לאתרים (`http_checker.sh`)

### מה זה עושה?

מקבל רשימת אתרים, שולח בקשת כותרים מהירה (`curl --head`) ומציג אינדיקציה ויזואלית ברורה:

- ✅ עבור סטטוס תקין או הפניה.
- ❌ עבור שגיאה.

הבדיקה מתבצעת לפי קוד ה-HTTP שחוזר מהשרת.

### איך מריצים?

```bash
./scripts/http_checker.sh
```

### קלט נדרש בזמן הריצה

רשימת כתובות אתרים מופרדים ברווחים.

דוגמה:

```text
google.com github.com
```

### הערה

הסקריפט מוודא שפקודת `curl` מותקנת במערכת.

במידה ולא, הוא יציע להתקין אותה אוטומטית (דורש הרשאות `sudo`).


---

### רז יעקבי
אחראי על כתיבת וניהול 5 הסקריפטים הבאים:

1. **גיבוי תכולה של תיקייה לקובץ tar.gz** (`backup_directory.sh`)
2. **ספירת שורות, מילים, ותווים בכל קובץ בתיקייה והדפסתם** (`word_count_directory.sh`)
3. **מחיקת קבצים הישנים יותר מ-X ימים, מתיקייה מסוימת** (`cleanup_old_files.sh`)
4. **טיימר ספירה לאחור, לפי משך זמן שהוגדר HH:MM:SS בפורמט** (`countdown_timer.sh`)
5. **ניקוי קבצים זמניים מפרויקט** (`clean_project.sh`) 


## 1. גיבוי תכולה של תיקייה לקובץ tar.gz (`backup_directory.sh`)

### מה זה עושה?
מגבה תיקייה שלמה ומכווץ אותה לקובץ ארכיון בסיומת `tar.gz`. הסקריפט מוסיף באופן אוטומטי חותמת זמן ייחודית (תאריך ושעה) לשם הקובץ הנוצר כדי למנוע דריסה של גיבויים קודמים.

### קלט נדרש בזמן הריצה:
* **ארגומנט 1:** נתיב לתיקיית המקור (התיקייה שאותה תרצו לגבות).
* **ארגומנט 2:** נתיב לתיקיית היעד (המיקום שבה יישמר קובץ הגיבוי).

### איך מריצים?
```bash
./scripts/backup_directory.sh /path/to/source_dir /path/to/backup_destination
```

## ספירת שורות, מילים ותווים בתיקייה (`word_count_directory.sh`)

### מה זה עושה?
סורק את כל הקבצים שנמצאים בתוך התיקייה שצוינה (ברמת התיקייה הראשית בלבד) ומציג בטרמינל פלט טבלאי מסודר של כמות השורות, המילים והתווים עבור כל קובץ בנפרד.

### קלט נדרש בזמן הריצה:
* **ארגומנט 1:** נתיב לתיקייה שאת הקבצים שלה תרצו לספור ולנתח.

### איך מריצים?
```bash
./scripts/word_count_directory.sh /path/to/target_directory
```




## מחיקת קבצים ישנים מתיקייה (`cleanup_old_files.sh`)

### מה זה עושה?
סורק תיקייה מוגדרת, מאתר ומציג את כל הקבצים שלא שונו או עודכנו במשך מספר ימים מסוים (X) ומעלה, ומוחק אותם לצמיתות כדי לפנות מקום בדיסק.

### קלט נדרש בזמן הריצה:
* **ארגומנט 1:** נתיב לתיקייה שבה תרצו לבצע את הניקוי.
* **ארגומנט 2:** מספר שלם המייצג את מספר הימים (למשל 30 ימים).

### איך מריצים?
```bash
./scripts/cleanup_old_files.sh /path/to/logs_directory 30
```



## טיימר ספירה לאחור (`countdown_timer.sh`)

### מה זה עושה?
מפעיל שעון עצר (טיימר) ויזואלי בטרמינל הסופר לאחור בזמן אמת על בסיס הזמן שהוגדר. הפלט מתעדכן על המסך באותה השורה ובסיום הספירה מושמע צליל התרעה של המערכת.

### קלט נדרש בזמן הריצה:
* **ארגומנט 1:** משך הזמן הרצוי לספירה לאחור בפורמט קשיח של `HH:MM:SS` (לדוגמה: `00:02:15` עבור שתי דקות ו-15 שניות).

### איך מריצים?
```bash
./scripts/countdown_timer.sh 00:02:15
```

## ניקוי קבצים זמניים מפרויקטים (`clean_project.sh`)

### מה זה עושה?
סורק ומנקה בצורה רקורסיבית (לעומק כל תתי-התיקיות) קבצים ותיקיות זמניים שנוצרים בזמן ריצה ופיתוח קוד, כגון תיקיות `node_modules`, תיקיות מטמון של פייתון (`__pycache__`, `*.pyc`), קבצי קומפילציה של ג'אווה (`*.class`) ותיקיות `build` זמניות.

### קלט נדרש בזמן הריצה:
* **ארגומנט 1:** נתיב לתיקיית השורש (Root) של פרויקט הפיתוח שאותו תרצו לנקות.

### איך מריצים?
```bash
./scripts/clean_project.sh /path/to/my_workspace/my_project
```
